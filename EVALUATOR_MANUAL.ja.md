# FACESIV 評価アプリ 使い方マニュアル（外部評価者向け）

本マニュアルは、臨床心理士や研究者などの外部評価者が、ロボット家族（エージェント）の会話ログをもとに「FACESIV（家族機能評価尺度）」で評価していただくにあたっての使い方の解説です。

---

## 1. はじめに：本評価アプリの目的

本評価アプリは、**「LLM（生成AI）がロボット家族の会話内容をどれくらい正確に解釈し、FACESIVに基づく評価ができているか」というLLMの評価の妥当性**を人間（専門家）の目から検証するためのツールです。

評価者の皆様には、画面上で「ロボット家族の会話履歴」を読み、その家族がいまどのような状態にあるかを FACESIVの全62項目（5段階評価）に沿って採点していただきます。

---

## 2. 評価アプリへのアクセス（環境構築）

本評価アプリはお使いのPC（Windows / Mac / Linux）のブラウザ（Google Chrome推奨）上で直接動かすことができます。
プログラミング等の専門的な知識は不要ですが、評価アプリを動かすための土台となるソフトウェアの準備が必要です。
以下の手順に沿って進めてください。

### 【STEP A】 土台づくり（Docker Desktop のインストール）

評価アプリを動かすためには、「Docker Desktop」という無料のソフトウェアが必要です。

1. **[Docker Desktop のダウンロードページ](https://www.docker.com/products/docker-desktop/)**にアクセスします。
2. お使いのパソコン（Windows なら「Download for Windows」、Macなら「Download for Mac」）に合わせて青いボタンをクリックし、ダウンロードします。
3. ダウンロードしたファイルを開き、画面の指示に従ってインストールしてください。（インストール中に専門用語が並ぶ画面が出ることがありますが、基本的には「OK」や「Next」を押し続けて大丈夫です）
4. インストールが完了したら、**Docker Desktop を起動**してください。
   * ※ 起動すると、右下のタスクトレイ（時計の横）や、Macの上のメニューバーに、クジラのアイコン🐳が表示されます。**このクジラアイコンが出ている（起動している）状態**にしておいてください。特に、PCを再起動した場合にDocker Desktopを起動し忘れていて、正しいコマンドを打っているのに反応しないといった勘違いが生じやすいです。

### 【STEP B】 黒い画面（ターミナル）を開く

ここから映画などでハッカーが使っているような黒い画面（ターミナル）を使いますが、指定の文字をコピー＆ペースト（貼り付け）するだけですのでご安心ください。

* **Windows の場合**
  * 画面下のスタートボタンを右クリックし、ターミナルもしくはコマンドプロンプトを開きます。
* **Mac の場合**
  * 画面右上の虫眼鏡マーク（Spotlight）をクリックし、「ターミナル」と入力して検索し、「ターミナル.app」を開きます。

### 【STEP C】 システムのダウンロードと設定ファイル（APIキー）の準備

開いた黒い画面（ターミナル）に、以下の文字を1行ずつコピーして貼り付け、**1行ごとにEnter（Return）キーを押して**ください。

**1. GitHubリポジトリ（評価アプリ込み）をPCにダウンロードする（Windowsの場合、ターミナルの今いる場所（ディレクトリ）は「C:\Users\ユーザ名」になっている）**
```bash
git clone https://github.com/robotaichi/rfs.git
```
※ 「`'git' は、内部コマンドまたは外部コマンド...として認識されていません`」といったエラーが出た場合は、今開いたターミナルを閉じてください（閉じないと次に行うインストール後の設定が反映されないため）。閉じた後に「Git for Windows」をhttps://gitforwindows.org/ からインストールしてください。これはWindowsでgitコマンドを扱えるようにするものです。インストール後にターミナルを起動し、再度上記のgit cloneコマンドを実行してください。

**2. ダウンロードしたフォルダに移動する（ディレクトリは「C:\Users\ユーザ名\rfs」となる）**
```bash
cd rfs
```
※ 画面の文字の左側（今いるディレクトリの場所）が `...rfs>` のように変わればOKです。

**3. APIキー（パスワードのようなもの）を設定する**
このシステムでは、ロボット家族（LLM）を動かすために「APIキー」という専用のパスワードが必要です。
評価アプリのみの使用であれば（LLMを使った会話生成などをしなければ）、実際にAPIの従量課金が発生することはないのでご安心ください。
APIキーはこちらから発行できます。
ChatGPT：https://platform.openai.com/api-keys
Gemini：https://aistudio.google.com/app/apikey

Mac / Linux の場合は以下のコマンドを実行します。
```bash
cp docker/.env.example .env
nano .env
```
※ `nano .env` を実行すると、画面が切り替わります。矢印キーでカーソルを動かし、`OPENAI_API_KEY=` と `GEMINI_API_KEY=` の右側に、それぞれ提供されたAPIキーを貼り付けてください（""などは不要でそのまま中身を貼り付ける）。APIキーは
※ 貼り付けたら、`Ctrl` キーを押しながら `X` を押し、次に `Y` を押し、最後に `Enter` を押すと保存されて元の画面に戻ります。

> **Windowsの場合**
> ターミナル操作が難しい場合は、パソコンの標準機能（エクスプローラー等）でダウンロードされた `rfs` フォルダの中にある `docker` フォルダを開いてください。その中の `.env.example` というファイルをコピーして、同じ場所に `.env` という名前に変更して保存（リネーム）してください。その後、その `.env` ファイルをメモ帳などで開いてAPIキーを書き込んで保存してください。

### 【STEP D】 システムの実行（起動）

ここから、いよいよ評価アプリをビルドして動かせるように環境構築を行います。
ターミナルに以下のコマンドを貼り付けてEnterキーを押してください（自動で環境構築されます）。

```bash
docker compose -f docker-compose.evaluator.yml up --build
```

* ビルドには **数分〜10分程度** かかります。
* 画面に英語のプログレスバーや文字がものすごい勢いで流れますが、**「エラー（Error）」で真っ赤になって止まらなければ正常**ですので、コーヒーを飲むなどしてそのまま気長にお待ちください。
* もし画面の最後に `Error` や `failed to connect to the docker daemon` などの文字が出て止まってしまった場合は、**Docker Desktop（クジラのアイコン）がきちんと起動しているか**を再確認してください。

### 【STEP E】 評価画面を開く

上記の画面で文字の流れが落ち着き（止まったように見え）、エラーが出ていなければ準備完了です！

PCのウェブブラウザ（Google Chrome推奨）を開き、アドレスバー（一番上のURLを入れるところ）に以下を入力してください。

> **アクセス先URL：** `http://localhost:5001/`

この画面が表示されたら、環境構築は成功です！
**ターミナルの黒い画面は開いたまま（裏でシステムが動いている状態）**にしておき、ブラウザのほうで以下の操作手順に進んでください。

---

### 【STEP F】 評価方法

評価アプリが立ち上がっているので、アプリの指示にしたがって評価をお願いします。
すべての評価が終わったら、裏で動かしっぱなしにしているターミナルに戻ります。
その画面上で、**キーボードの `Ctrl` キーを押しながら `C` キーを押して**ください。
評価アプリが安全に終了します。（ターミナル自体も閉じてしまって構いません）

## 3. 画面の見方と操作手順

評価の実施は、大きく以下の **4つのステップ** で進行します。

### ステップ 1：評価対象（アーカイブ）の選択

評価アプリにアクセスすると、「Select an Archive」という画面が表示されます。
ここには、過去に実施された実験や会話の記録（アーカイブ）が一覧で並んでいます。

1. **評価を依頼されたアーカイブ（例：`experiment_202X...`）をクリック**してください。
2. クリックすると、そのアーカイブに含まれる複数のセッション（会話の区切り）の評価画面に自動で移動します。

---

### ステップ 2：評価者名の入力と会話の確認

画面が「FACES-IV Questionnaire（評価画面）」に切り替わります。画面は左右2つのエリアに分かれています。

#### ① 評価者名の入力（画面上部）
まずは、**「👤 Evaluator Name:」** の入力欄にあなたのお名前（または指示されたID等）を入力してください。この名前は結果を保存する際に使用されます。

#### ② 会話内容を読む（画面右側：Conversation）
画面の右側には、対象となるセッションでの「ロボット家族の実際の会話履歴（チャット形式）」が表示されています。
* スクロールして過去の発言を遡ることができます。
* この会話から読み取れるろボット家族のコミュニケーションや関係性を把握してください。
* 会話履歴は常に右側に表示されているので質問項目ごとに確認していただいて構いません。

---

### ステップ 3：FACESIV の採点（画面左側）

画面左側には、FACESIV の質問項目（全62問）が並んでいます。
右側の会話を読んだ印象に基づき、対象となる家族（またはシステムとのやり取り）が以下のどれに該当するか、ラジオボタンをクリックして選択してください。

* **1** = 全く当てはまらない (Strongly Disagree)
* **2** = あまり当てはまらない (Generally Disagree)
* **3** = どちらともいえない (Undecided)
* **4** = やや当てはまる (Generally Agree)
* **5** = 非常に当てはまる (Strongly Agree)

> **💡 ヒント**
> 回答を選択すると、その行の色が変わり、「回答済み」であることが視覚的にわかるようになっています。

#### セッションの移動
すべての項目（または評価可能な項目）に回答し終えたら、右下にある **「Next Session →」** ボタンを押して、次のセッション（別の会話）の評価へ進みます。
* 未回答の項目がある場合は、警告が表示されますが、そのまま進むことも可能です（未回答の項目はハイライトされます）。
* 「← Previous Session」ボタンで前のセッションに戻って回答を修正することもできます。

---

### ステップ 4：結果の確認と保存（最終画面）

すべてのセッションの評価が完了し、最後のセッションで **「📊 View Results (結果を見る)」** ボタンを押すと、「結果確認画面（Score Comparison）」に移動します。

#### ① 評価結果の比較
この画面では、あなたが付けた点数（Evaluator列）と、LLMが採点した点数と平均点（Mean列）、およびその差異（Diff列）が表形式で確認できます。
* 差異が小さい項目、大きい項目などが色分け等で示されます。
* 画面上部のタブ（Session 1, Session 2...）を切り替えることで、各セッションごとの結果を見られます。
* 「← Back to Evaluation」ボタンで評価画面に戻ることも可能です。

#### ② 結果の保存（重要）
評価内容をシステム上に記録するため、最後に必ず画面下の **「💾 Save as CSV」** ボタンをクリックしてください。
* ボタンの下に **「✅ Saved successfully!」** と表示されれば完了です。
* これにより、あなたの評価データが手元に安全に保存されます（保存先が表示されるのでご確認ください）。

以上で評価作業は完了です。ブラウザのタブを閉じて終了してください。

## 4. よくある質問・注意事項

* **途中でブラウザを閉じても大丈夫ですか？**
  * いいえ。セッションを移動した際の一時的な記憶はされていますが、最後まで進んで「Save as CSV」を押す前にブラウザを完全に閉じてしまうと、入力した内容が消えてしまう可能性があります。可能な限り、一連の回答を完了させてから終了してください。
* **会話データが表示されません。**
  * エラーなどの理由で会話データが存在しないセッションの場合があります。その場合は管理者に連絡するか、スキップ（回答せずに Next Session へ）してください。
* **スマートフォンからでも回答できますか？**
  * 基本的にはパソコン（または画面の大きなタブレット）の横画面での利用を想定してレイアウトされています（左に質問、右に会話）。スマートフォンでも操作は可能ですが、表示が崩れる等の見にくさが発生する場合があります。
